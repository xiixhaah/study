1、项目中的权限控制具体做了哪几层，如果一个用户没有权限去访问页面的话怎么拦截
维度：路由页面（视图层）、页面层（UI层）、网络层（后端接口层 防止用户绕过前端去调用API  最重要的防线）、数据层（决定了用户访问数据的范围）  前端不能直接做权限控制
前后端：前端-路由守卫（静态路由、动态路由） 登陆成功后获取到权限列表，路由判断，返回页面
后端-安全 健全中心件 JWT 双Token机制 来获取用户身份； 异常相应状态码，前端弹出异常信息
避坑点：要说前后端



在项目中，权限控制我们是前后端协作完成的，核心思路是 “前端防君子、后端防小人”，因为前端的控制可以被绕过，所以后端是安全的核心防线。具体分为以下四层：
1. 路由页面（视图层）控制
前端在用户登录成功后，会从后端获取该用户的权限列表。我们采用静态路由 + 动态路由的方案：
    静态路由：登录页、404 页等无需权限的页面直接开放；
    动态路由：根据权限列表动态生成可访问的路由表，通过路由守卫进行拦截 —— 如果用户尝试访问无权限的路由，直接跳转到无权限提示页或首页。
2. 页面层（UI 层）控制
前端在页面渲染时，会根据权限控制按钮、表单等 UI 元素的显示 / 隐藏 / 禁用（比如没有 “编辑” 权限就隐藏编辑按钮）。这层主要是优化用户体验，避免误操作，但不做安全核心。
3. 网络层（后端接口层）控制【最重要防线】
这是防止用户绕过前端直接调用 API 的核心，后端通过以下方式保障：
    身份认证：用JWT 双 Token 机制（Access Token + Refresh Token），用户请求接口时必须携带 Token，后端通过安全中间件解析 Token 确认身份；
    接口权限校验：中间件确认身份后，进一步校验用户是否有该接口的访问权限；
    异常响应：如果无权限，后端返回 403 状态码，前端统一拦截并弹出 “无权限访问” 的提示。
4. 数据层控制
后端在业务逻辑层，根据用户权限限制数据访问范围（比如普通用户只能查自己的数据，管理员能查全量数据），从根源上防止越权获取数据。

总结来说，前端做路由和 UI 控制是为了体验，后端的接口 + 数据层控制才是安全核心，前后端配合才能形成完整的权限闭环。
