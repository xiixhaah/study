# 行为场景题（非技术类）：过往经历
技术黑洞：面试官，请稍等我思考实际十几s。可以问面试官是关于哪个技术的，不要说不知道
拖延时间 可以问是哪个技术点

具体例子

1、团队冲突经历以及怎么解决：技术方案和业务
根源（技术偏向） 彼此的争论拉回用户体验上
用数据进行说话 埋点、日志、对比
领导拍板决策
反思：对业务或技术上的反思

2、学习能力
例子+方法+反思
为什么学习这个技术
学习方法
跟同事之间进行有效分享

3、抗压表现
时间管理 压力管理
抗压：睡眠 交流沟通 运动 怎么推进（todolist）

4、最大的缺点
致命伤（拖延、迟到、抗压、情绪） 缺点+改进
过渡的追求细节：对项目、文档等过渡整洁，可能忽略速度，意识到效率和质量的平衡，采取灵活的方式
不太擅长拒绝别人，安排太满，导致作息被压缩，完成自己该做的任务，根据优先级完成任务，学会拒绝

5、领导力（统筹力）
对齐目标：任务 达到效果 人力资源分工 时间分配方案 
反思：技术选型方案 大的技术方向 如何去确定一个技术的规范文档 评审机制和规范等
